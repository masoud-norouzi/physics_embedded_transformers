\subsection{Future Scientific Role}

The physics-state package is a placeholder for a unified interface between the
physics engine and the ML models. In future phases it should expose synchronized
state variables such as geometry coordinates, droplet occupancy, branch
hydraulic features, CFD velocity lookups, and possibly streamline-derived
features.

No functionality is implemented in this package yet. This is intentional: the
architecture reserves a stable location without prematurely defining an API.
