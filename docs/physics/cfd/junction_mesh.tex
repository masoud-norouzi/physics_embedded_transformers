\subsection{Physical Significance}

The CFD module begins the transition from lumped branch resistance to
spatially resolved flow. Phase 1 constructs a computational domain and mesh for
the upper junction. No CFD solve is implemented yet.

The future physical problem will be low-Reynolds-number flow in a confined
microchannel. For an incompressible Newtonian fluid, the eventual Stokes
equations would be:

\[
-\nabla p + \mu \nabla^2 \mathbf{u} = \mathbf{0},
\qquad
\nabla \cdot \mathbf{u} = 0.
\]

Those equations are not solved in the current phase. The current CFD package
only prepares the domain and mesh required for such a solver.

\subsection{Domain Construction}

The domain is extracted from the upper junction topology:

\[
\mathrm{inlet}
\rightarrow
\{\mathrm{left~branch}, \mathrm{right~branch}\}.
\]

The computational patch extends \(400~\mu\mathrm{m}\) along the inlet and
\(400~\mu\mathrm{m}\) downstream along each branch. Coordinates are stored in
micrometers throughout:

\[
\mathbf{x}_{\mu\mathrm{m}} = \Delta x \mathbf{x}_{\mathrm{px}},
\qquad
\Delta x = 4~\mu\mathrm{m}/\mathrm{px}.
\]

The idealized channel domain is the set of points within half a channel width
of the extracted branch centerlines:

\[
\Omega_{\mathrm{CFD}}
=
\left\{
\mathbf{x} :
\min_{\mathbf{c}\in \Gamma}
\|\mathbf{x}-\mathbf{c}\|_2
\le
\frac{W}{2}
\right\},
\qquad
W=100~\mu\mathrm{m}.
\]

Here \(\Gamma\) is the union of the inlet, left, and right branch centerline
segments in the junction patch.

\subsection{Mesh Generation}

The mesh is triangular and generated from a triangular lattice point cloud
filtered against the idealized channel domain. This avoids pixelated boundaries
and avoids adding external meshing dependencies at this stage. Mesh sizing is
driven entirely by:

\begin{verbatim}

\end{verbatim}

\subsection{Mesh Quality}

For each triangular element, side lengths are \(a,b,c\). Angles are computed by
the law of cosines:

\[
\theta_a
=
\cos^{-1}
\left(
\frac{b^2+c^2-a^2}{2bc}
\right).
\]

Element area is:

\[
A_e
=
\frac{1}{2}
\left|
(\mathbf{x}_2-\mathbf{x}_1)
\times
(\mathbf{x}_3-\mathbf{x}_1)
\right|.
\]

The aspect ratio is reported as:

\[
\mathrm{AR}_e =
\frac{\max(a,b,c)}{\min(a,b,c)}.
\]

\subsection{Implementation Details}

\texttt{domain.py} owns CFD-specific junction geometry extraction. Generic
device centerline utilities remain under \texttt{src/physics/geometry}. This
keeps reusable geometry separate from CFD-specific domain construction.

\texttt{mesh.py} provides:

\begin{verbatim}
geometry = build_junction_geometry(config)
mesh = generate_mesh(geometry)
report = evaluate_mesh(mesh)
\end{verbatim}

\texttt{solver.py}, \texttt{streamlines.py}, and \texttt{velocity\_field.py}
are placeholders only. They intentionally contain no Stokes solver, pressure
field, streamline extraction, or velocity lookup logic.
